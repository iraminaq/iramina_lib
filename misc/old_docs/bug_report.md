# MergeSortTree のテストに関するバグ報告

`test\MergeSortTree\test.cpp` をコンパイル・実行したところ、アサーションエラーが発生しました。

## エラー内容
```
Assertion failed: count_less
got      = 4
expected = 5
```

## 原因
`test.cpp` 内の `test_small_fixed` 関数における期待値（expected）の設定に誤りがあると考えられます。

テストで与えられている配列 `a` は以下の通りです。
`a = {5, 1, 4, 1, 3, 9, 2, 6}`

この配列全体の区間 `[0, 8)` に対して、値 `4` 未満の要素（`count_less`）は `{1, 1, 3, 2}` の **4個** です。
しかし、テストコード側では期待値が `5` と設定されています。

同様に、その後続するアサーションの期待値もずれている可能性があります。
* `count_less(0, 8, 4LL)` の実測値は `4` (期待値 `5` になっておりエラー)
* `count_less_equal(0, 8, 4LL)` の実測値は `5` (期待値 `6` に設定されている)

**結論:** `MergeSortTree` 側の実装のバグではなく、`test.cpp` 側の期待される値の設定（テストケース自体）が誤っているバグです。指示の通りソースコードやテストコードの修正は行わず、こちらに原因をまとめました。
